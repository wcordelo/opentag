# OpenTag 2 gap remediation report

Date: 2026-07-16. Scope: the 23 findings in `goal-outputs/opentag-2-gap-audit/gap-audit.md`, evaluated against the current shared worktree after the final corrective and validation pass. This report describes source and local-test state only. It does not treat an available primitive as a resolution unless the production call site uses it and a regression test exercises the relevant boundary.

## Resolution summary

All 23 original findings have a source-level resolution or, for M1, an explicit and tested implementation choice that satisfies the audit's allowed “chosen equivalent” disposition. The final census is 4 Critical, 8 High, 8 Medium, and 3 Low findings, with no omitted ID. Corrective attempt 4 closed the last C2, H3, M3, and L2 review gaps: recoverable answer pages are decoration-free and byte-for-byte stable across normal AG-UI and alarm recovery; staging failures at either size tier abort for exact-byte retry; delayed-file correlation retains multiple immutable pending mentions and refuses ambiguity; SessionEvent replay rejection starts neither runtime and releases no final lifecycle ownership; and canonical docs now require SessionEvent input/output/tool data to exist before Slack delivery.

The highest-risk invariants are now enforced at the boundary where they matter. Repository coding and every selected-Claude turn fail visibly when the authoritative harness is unavailable instead of reaching AG-UI. A live Slack create reserves a stable client identity, durably confirms the returned timestamp, and performs a bounded exact lookup after an ambiguous create. Every recoverable answer page is built only from the canonical answer text and page index, with no session-link or artifact-card decoration. Session and card decoration is emitted as a separate stable-ID message, so a decorated normal AG-UI turn and alarm recovery still send the exact same answer-page text, blocks, thread, page boundary, and client ID after an applied-response-loss fault.

Stop, delivery, session, and interaction control now use exact identities throughout. Request-time and alarm-resumed AG-UI Stop require matching accepted and quiescent proof; harness and research controls have their own exact barriers. SessionEventDO multi-row transitions use `transactionSync`. Complete immutable quick-click, delayed-file, and ordinary file-turn jobs are stored in `DEFERRED_INGRESS`; a duplicate nonterminal job repairs any missing alarm at its preserved retry deadline before Slack receives HTTP 200, then the alarm replays through authenticated `BOT_SELF` with bounded retry and retained exhaustion evidence. Bot and research Slack writes reserve the same cross-script per-channel `SLACK_RATE_LIMIT` owner for every HTTP attempt, including identical-form `Retry-After` replays; an exhausted research 429 remains ambiguous and drainable. The render-obligation alarm remains a separate durable sequential owner.

Attachment and context handling now covers the audited path end to end: bounded inline and staged tiers; fail-closed R2 staging at every size when a stager is configured; durable file-turn and delayed-file repair ownership; append-only unique pending mentions scoped by team/channel/user and event ID; exact-thread selection, ambiguity refusal, and consumed markers; rich history normalization; and structured SessionEvent input, output, and tool summaries. A failed R2 copy, SessionEvent append, or SessionEvent replay stops runtime/final progress rather than advancing noncanonical state. Successful replay converts canonical attachment metadata into staged prompt parts and sends those attachments plus the reconstructed tool transcript through the real authenticated harness path.

No production configuration or remote system was changed during remediation. The remaining work is activation and live verification, listed separately below; those gates are not being represented as completed source work.

## Remaining limitations

The original 23 audit gaps are addressed in source, but several deliberate product/operations limits remain:

- The authoritative harness is source-complete but not enabled by default. With no approved harness activation, repository-coding and selected-Claude turns fail visibly; they do not fall back. This is a safety property, not a usable production coding surface until configured.
- Harness NDJSON is durably recorded, but the Slack coding response is currently posted as one final fenced answer rather than live coding deltas. The `onText` hook remains the future extension point.
- The alternate harness matrix is intentionally narrow: Claude Code is supported; Codex, Amp, and generic reasoning flags are not advertised as active runtimes.
- Staged attachments depend on a shared R2 namespace and access to the original Slack file. When a stager is configured, both inline-sized and staged-sized delivery fail closed if the durable copy cannot be made; the harness frontend also enforces a 32 MiB resolved aggregate cap and fails closed on missing objects, size mismatch, digest mismatch, or absent `BLOBS`.
- Session links and artifact cards are intentionally a separate Slack message from recoverable answer pages. This preserves exact alarm replay identity/body at the cost of an additional stable-ID message on decorated final answers.
- SessionEvent append and replay failures intentionally suppress runtime/final delivery and retain lifecycle ownership. Operators must restore the canonical store and allow retry rather than expecting an AG-UI/harness fallback or a best-effort final post.
- Bot and research Slack rate scheduling is cross-isolate/cross-script only when their operator-deployed configs bind the same bot-owned `SLACK_RATE_LIMIT` instance. Bounded research 429 exhaustion intentionally remains ambiguous and drainable instead of being treated as a definitive delivery failure. Render-obligation alarm recovery remains a separate sequential Durable Object owner.
- Quick clicks, delayed-file repairs, and ordinary file-bearing turns depend on operator-deployed `DEFERRED_INGRESS` and `BOT_SELF`. Identical Slack retries repair a missing alarm on a stored nonterminal job, but this run did not activate those bindings in production.
- The session viewer is a signed, expiring, read-only JSON surface. It is not a full operator console and does not expose mutation controls.
- Research action cards create ordinary synthetic turns; whether “Export” can produce a particular artifact still depends on the tools and policies available to that subsequent agent turn.
- The harness outbound policy is deliberately GitHub-specific. Other git hosts require their own canonical URL parser, allowlist, credential boundary, branch proof, and API authorization implementation.
- Local fault injection and mock Slack/API tests cannot prove account permissions, Cloudflare service-binding names, R2 bucket identity, Slack app installation, or GitHub token scopes. Those are the reasons the activation checklist remains separate from the source-resolution matrix.

No production traffic or external state is used as evidence in this report. The durable deliverable is the current source, tests, validation script, and this finding-by-finding record.
